## 2. 인증 및 접근 제어

### 2-1. 스태프 로그인
- [ ] `/login` 접속 → Staff 탭 선택 → 아이디/비밀번호 폼 표시
- [ ] 올바른 계정 입력 → 로그인 성공 → `/admin` 이동
- [ ] 잘못된 비밀번호 → 에러 메시지 표시
- [ ] 5회 연속 실패 → 15분 계정 잠금 메시지

### 2-2. 카카오 로그인 (재원생)
- [ ] `/login` 접속 → 카카오 로그인 → OAuth callback → `/register` 또는 홈 이동
- [ ] 최초 가입 시 → `/register`에서 추가 정보 입력 → 가입 완료

### 2-3. 접근 제어 — 비로그인
- [ ] `/notices` 접근 → `/login?redirect=/notices`로 리다이렉트
- [ ] `/questions` 접근 → `/login`으로 리다이렉트
- [ ] `/admin` 접근 → `/login`으로 리다이렉트
- [ ] `/my` 접근 → `/login`으로 리다이렉트

### 2-4. 접근 제어 — 역할별
- [ ] student 계정 → `/admin` 접근 → `/`로 리다이렉트
- [ ] assistant 계정 → `/admin/guide`만 접근 가능, 나머지 `/admin/*` → `/`로 리다이렉트
- [ ] mentor 계정 → `/admin/*` 전체 접근 가능
- [ ] admin 계정 → 전체 접근 가능
- [ ] staff 계정 → `/meal` 접근 → admin이면 `/admin`, 아니면 `/`로 리다이렉트

### 2-5. 접근 제어 — 계정 상태 (재원생)
- [ ] pending 상태 재원생 → `/notices` 접근 → `/pending-approval`로 리다이렉트
- [ ] inactive 상태 재원생 → `/notices` 접근 → `/account-inactive`로 리다이렉트
- [ ] active 상태가 아닌데 `/pending-approval` 접근 → `/`로 리다이렉트 (상태 불일치)

### 2-6. 로그아웃
- [ ] 어드민 사이드바 로그아웃 클릭 → Supabase 세션 해제 + Zustand 초기화 + `/`로 전체 리로드
