## 3. 요구사항

### 3.1 도시락 신청 (재원생)

**기능:**
- 현재 활성화된 신청 기간 조회
- 요일별/날짜별 중식/석식 선택
- 신청 내역 확인 및 수정
- 신청 기간 외에는 조회만 가능

**제약:**
- `active` 상태 재원생만 신청 가능
- 신청 기간 내에만 수정 가능

### 3.2 출석 체크

**출석 방식 (우선순위):**
1. **QR 코드** - 학원 내 QR 스캔 (권장)
2. **PIN 코드** - 4자리 숫자 입력 (백업)

**관리자 기능:**
- 오늘 출석 현황 대시보드
- 출석/퇴실 시간 조회
- 수동 출석 처리
- 출석 통계 리포트

**재원생 기능:**
- QR 스캔 또는 PIN 입력으로 출석
- 본인 출석 이력 조회
- 출석 통계 (월별)

## 4. 기술적 결정

### 4.1 QR 코드 생성

- 관리자가 매일 새 QR 코드 생성 (또는 자동 갱신)
- QR 내용: `studycore://checkin/{daily_code}`
- `daily_code`: 매일 자정에 자동 갱신되는 6자리 코드

### 4.2 위치 검증 (선택)

- Geolocation API로 학원 반경 내 확인
- 정확도 이슈로 1차 버전에서는 생략 가능

### 4.3 중복 출석 방지

- 같은 날 같은 타입(check_in/check_out) 중복 불가
- 출석 후 퇴실 처리 시에만 check_out 가능
